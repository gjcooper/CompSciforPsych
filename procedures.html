    API Design

    Web API's

    • SOAP
    • REST
    • json and xml
    • Simple Object Access Protocol - Heavyweight, Passing XML
    • Representational State Tansfer - Based over HTTP, Most likely to be consuming this
    • Extensible Markup Language, Javascript Object Notation

    Library API

    • Small scope
    • Try not to change it often
    • Document what is part and what is not part of it
    • Can evolve, don't try to cover every use case
    • Version number contract
    • Document it well

    JOSS


    Image courtesy of Pixabay
    • You've made some decent software
    • Now profit!!

    What is it?

    • An academic journal (with ISSN etc)
    • Goal is to be able to submit a software package to JOSS in under 1 hour

    How???

    • Software available in open repository, using approved open source license
    • Author a short Markdown papaer with title, summary, author names, affiliations, and key references
    • Ideally add a short metadata file to your repository

    It can't be that simple?

    Well not really

    • You need to be a major contributor to the software
    • Software needs good API documentation
    • Software needs good End User documentation
    • A way of testing the functionality of the code needs to be implemented
    • Community Guidelines need to be in place
    • Should be feature complete
